# File/Image Upload Template — Page & Component Design (Desktop-first)

## Global Styles (ใช้ร่วมทั้งแอป)
- Design tokens (แนะนำ):
  - Background: ขาว/เทาอ่อน
  - Primary: สีหลักของระบบ
  - Border: เทากลาง (เส้น 1px)
  - Radius: 8px (การ์ด/กล่อง)
  - Typography: ขนาดฐาน 14–16px, หัวข้อ 18–20px
- ปุ่ม/ลิงก์:
  - Primary button: พื้นหลังสีหลัก, hover เข้มขึ้น
  - Link: ขีดเส้นใต้เมื่อ hover

## Page Meta (สำหรับ “หน้าฟอร์มที่มีการแนบไฟล์/รูป”)
- Title: ตามชื่อฟอร์ม + “แนบไฟล์” (เช่น “สร้างคำขอ – แนบไฟล์”)
- Description: “กรอกข้อมูลและแนบไฟล์/รูปประกอบ”
- Open Graph: ใช้ค่าเริ่มต้นของระบบ (ชื่อฟอร์ม + คำอธิบายสั้น)

## Page Structure (หน้าฟอร์ม)
- โครงหลักแบบ stacked sections (desktop-first):
  1) Header ของฟอร์ม (ชื่อฟอร์ม/คำอธิบายสั้น)
  2) กลุ่มฟิลด์ข้อมูล
  3) กลุ่ม “แนบไฟล์/รูป” (ใช้คอมโพเนนต์ส่วนกลาง)
  4) ปุ่มท้ายฟอร์ม (บันทึก/ส่ง)
- Layout: ใช้ Flexbox + ช่องไฟมาตรฐาน (เช่น 16–24px ระหว่าง section)

## Shared Component Spec: “File/Image Upload Field”
คอมโพเนนต์ส่วนกลางที่ทุกหน้าฟอร์มเรียกใช้ร่วมกัน เพื่อให้ UX สม่ำเสมอ (คลิกเลือกไฟล์ + ลากวาง, แสดงรายการ/พรีวิว, รองรับหลายไฟล์)

### 1) Layout
- Container แบบ card/boxed block กว้างเต็มคอลัมน์ของฟอร์ม
- ภายในแบ่ง 2 ส่วน (แนวตั้ง):
  1) Dropzone/Choose area
  2) File list + image previews

### 2) Sections & Components
#### 2.1 Label + Helper
- Label ฟิลด์: “แนบไฟล์/รูป”
- Helper text สั้น: “คลิกเพื่อเลือกไฟล์ หรือ ลากไฟล์มาวาง”

#### 2.2 Dropzone / Click-to-select Area
- พื้นที่สี่เหลี่ยมเส้นประ (dashed) สูง ~120–160px
- องค์ประกอบในพื้นที่:
  - ไอคอน (อัปโหลด)
  - ข้อความหลัก: “ลากไฟล์มาวางที่นี่”
  - ข้อความรอง: “หรือคลิกเพื่อเลือกไฟล์”
- Interaction:
  - คลิก: เปิด file picker
  - ลากไฟล์เข้ามาเหนือพื้นที่: เปลี่ยน state เป็น “พร้อมรับไฟล์” (เช่น เปลี่ยนสีขอบ)
  - วางไฟล์: เพิ่มไฟล์เข้า “รายการไฟล์”

#### 2.3 File List
- แสดงเป็นรายการแนวตั้ง (1 ไฟล์ = 1 แถว) หรือเป็นกริดการ์ดเล็กเมื่อเป็นรูปจำนวนมาก (ขึ้นกับบริบทของฟอร์ม)
- แต่ละรายการควรมี:
  - ชื่อไฟล์
  - ข้อมูลจำเป็นเพื่อยืนยัน (เช่น ขนาดไฟล์ ถ้าต้องแสดงใน UI ของแอป)

#### 2.4 Image Preview (เฉพาะไฟล์รูป)
- Thumbnail ขนาดคงที่ (เช่น 56–72px) จัดชิดซ้ายของรายการไฟล์
- คลิกที่รูป (ถ้าฟอร์มต้องการ): เปิดดูภาพขยายใน modal/overlay (หากแอปมีแพทเทิร์นนี้อยู่แล้ว)

### 3) States
- Default: ยังไม่มีไฟล์ → แสดง dropzone เด่น
- Hover/Drag-over: เน้นขอบ/พื้นหลัง
- Has files: dropzone ยังอยู่ แต่ลดความเด่นลง และเน้นรายการไฟล์

### 4) Responsive behavior
- Desktop-first:
  - หน้าฟอร์มใช้คอลัมน์เดียว (หรือ 2 คอลัมน์สำหรับฟิลด์อื่น) แต่บล็อกอัปโหลดควรเต็มความกว้างคอนเทนต์
- Mobile:
  - Dropzone เต็มความกว้าง, รายการไฟล์เป็นแนวตั้ง, thumbnail ลดขนาดเล็กลง

### 5) Centralization (จุดรวมศูนย์คอมโพเนนต์)
- ให้กำหนดเป็น “คอมโพเนนต์ฟอร์มมาตรฐาน” 1 ตัว เพื่อให้ทุกหน้าฟอร์มเรียกใช้รูปแบบเดียวกัน
- พฤติกรรมสำคัญต้องเหมือนกันทุกหน้า:
  - คลิกเลือกไฟล์ + ลากวาง
  - รองรับหลายไฟล์
  - แสดงรายการไฟล์ และพรีวิวรูป
