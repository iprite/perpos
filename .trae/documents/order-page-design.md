# Page Design Spec: หน้า “ออเดอร์” (Desktop-first)

## 1) Layout
- โครงหลัก: 2 คอลัมน์ (ซ้าย = เมนู/นำทาง, ขวา = เนื้อหา) ด้วย CSS Grid
  - Grid: `grid-template-columns: 240px 1fr; gap: 24px;`
- ภายในพื้นที่เนื้อหา: ใช้ Flexbox สำหรับจัดหัวข้อ/ปุ่ม/ฟอร์ม และตารางรายการบริการ
- Responsive (แนวทาง):
  - ≥1024px: แสดง 2 คอลัมน์ตามปกติ
  - <1024px: ซ่อนเมนูซ้ายเป็น Drawer และคง layout เนื้อหาเป็นคอลัมน์เดียว

## 2) Meta Information
- Title: ออเดอร์
- Description: สร้างออเดอร์ เลือกลูกค้า เพิ่มหลายบริการ และคำนวณส่วนลด/VAT
- Open Graph:
  - og:title: ออเดอร์
  - og:description: จัดการออเดอร์บริการ พร้อมส่วนลดและ VAT

## 3) Global Styles (Design Tokens)
- Background: `#F7F8FA`
- Surface (Card): `#FFFFFF`
- Text primary: `#111827`, secondary: `#6B7280`
- Primary color: `#2563EB` (ปุ่มหลัก)
- Danger: `#DC2626` (ลบบรรทัดบริการ)
- Radius: 10px (card), 8px (input/button)
- Typography:
  - H1 24/32 semibold, H2 18/28 semibold, body 14/22
- Buttons:
  - Primary: พื้นหลัง primary + ตัวอักษรขาว, hover เข้มขึ้นเล็กน้อย
  - Secondary: พื้นหลังขาว + เส้นขอบเทา
- Inputs:
  - สูง 40px, padding 10–12px, focus ring primary

## 4) Page Structure
หน้าเดียวแบบ “List + Create”
- ส่วนบน: ชื่อหน้า + actions
- ส่วนกลาง: รายการออเดอร์ (ตาราง/ลิสต์)
- ส่วนล่าง/ด้านขวา (ตามพื้นที่): พื้นที่ฟอร์มเพิ่มออเดอร์ (ค่าเริ่มต้น “ซ่อน”)
  - เมื่อกด “เพิ่มออเดอร์” ให้แสดงเป็น Drawer/Side Panel (แนะนำ) หรือ Inline Card

## 5) Sections & Components

### 5.1 Navigation (เมนู)
- รายการเมนูเดิมให้ “เปลี่ยนชื่อ” เป็น **ออเดอร์**
- สถานะ active เมื่ออยู่ route `/orders`

### 5.2 Header Bar
- ซ้าย: H1 “ออเดอร์”
- ขวา: ปุ่ม Primary “เพิ่มออเดอร์”
  - คลิกแล้ว: แสดงฟอร์มเพิ่มออเดอร์ (จาก hidden → visible)

### 5.3 Order List (รายการออเดอร์)
- Card ตารางรายการ (อย่างน้อยมีคอลัมน์): เลขที่/วันที่/ลูกค้า/ยอดรวม/สถานะ(ถ้ามีอยู่แล้ว)
- แถวในตาราง: คลิกเพื่อโฟกัส/เปิดดู (ถ้าระบบเดิมมีการดู/แก้ไข ให้คงพฤติกรรมเดิม)

### 5.4 Create Order Panel (ฟอร์มเพิ่มออเดอร์ — ซ่อนจนกดเพิ่ม)
**สถานะ**
- Default: ไม่แสดง panel และไม่มีฟอร์มรบกวนหน้า list
- Visible: แสดง panel พร้อมปุ่ม “ปิด” (X) และ “บันทึก”

**โครงฟอร์ม (เรียงจากบนลงล่าง)**
1) ข้อมูลลูกค้า
- Field: “ลูกค้า” (Customer Selector)
  - รูปแบบ: Combobox + search
  - แสดงผลที่เลือก: ชื่อลูกค้า (และข้อมูลประกอบที่มี เช่น เบอร์/รหัส ถ้ามีอยู่แล้วในระบบ)

2) รายการบริการในออเดอร์ (หลายบริการ)
- ตารางรายการบรรทัดบริการ (Line Items)
  - คอลัมน์แนะนำ: บริการ, รายละเอียด(ถ้ามี), จำนวน, ราคาต่อหน่วย, รวม, ปุ่มลบ
- ปุ่ม Secondary: “เพิ่มบริการ”
  - คลิกแล้วเพิ่มบรรทัดว่าง 1 บรรทัด
- Validation ขั้นต่ำ:
  - ต้องมีลูกค้า
  - ต้องมีอย่างน้อย 1 บริการ

3) สรุปราคา (ส่วนลด + VAT แบบใบเสนอราคา)
- แสดงบล็อกสรุปด้านล่างของตารางบริการ
- แถวสรุป (อย่างน้อย):
  - ยอดรวมก่อนส่วนลด
  - ส่วนลด (Input)
  - ยอดหลังส่วนลด
  - VAT (Input)
  - ยอดสุทธิ
- หลักการคำนวณ:
  - ใช้ “ลำดับและรูปแบบเดียวกับใบเสนอราคาในระบบ” (เพื่อให้ผลลัพธ์ตรงกัน)

4) Actions
- ปุ่ม Primary: “บันทึกออเดอร์”
- ปุ่ม Secondary: “ยกเลิก” (ปิด panel และเคลียร์ค่า ถ้าระบบเดิมทำเช่นนั้น)
- Loading state ระหว่างบันทึก: ปิดการกดซ้ำ + แสดง spinner ในปุ่ม

## 6) Interaction States
- เมื่อกด “เพิ่มออเดอร์”: โฟกัสไปที่ช่องเลือกลูกค้า
- เมื่อเพิ่ม/ลบบริการ: ยอดรวม/ส่วนลด/VAT รีคำนวณทันที
- Error state: แสดงข้อความใต้ field ที่ผิด + highlight ขอบแดง

## 7) Accessibility
- ปุ่ม/ช่องกรอกทุกตัวต้องมี label ชัดเจน
- Panel แบบ Drawer ต้อง trap focus และปิดด้วย Esc ได้
