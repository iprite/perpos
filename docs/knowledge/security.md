# ความปลอดภัยของระบบ PERPOS

## PERPOS ปลอดภัยแค่ไหน

PERPOS ออกแบบความปลอดภัยแบบหลายชั้น ตั้งแต่การเข้าสู่ระบบ การแยกข้อมูลรายผู้ใช้/องค์กร การเข้ารหัสข้อมูล ไปจนถึงการบันทึกร่องรอยการใช้งาน (audit log) เพื่อปกป้องข้อมูลธุรกิจและข้อมูลส่วนบุคคลของลูกค้า

## การเข้าสู่ระบบ (Authentication)

PERPOS ใช้การล็อกอินผ่าน **LINE Login** เป็นช่องทางหลัก จึงไม่ต้องตั้งและจดจำรหัสผ่านแยก ลดความเสี่ยงรหัสผ่านรั่วไหล การยืนยันตัวตนอาศัยบัญชี LINE ที่ผู้ใช้มีอยู่แล้ว ระบบยืนยันตัวตนผ่านมาตรฐาน OAuth

## การแยกและคุ้มครองข้อมูล (Row Level Security)

ทุกตารางข้อมูลใน PERPOS เปิดใช้ **Row Level Security (RLS)** บนฐานข้อมูล PostgreSQL (Supabase) ทำให้ผู้ใช้แต่ละคน/แต่ละองค์กรเข้าถึงได้เฉพาะข้อมูลของตนเองเท่านั้น ข้อมูลขององค์กรหนึ่งไม่สามารถถูกอ่านโดยอีกองค์กรได้ การเข้าถึงข้อมูลระดับระบบจำกัดเฉพาะ service role ฝั่งเซิร์ฟเวอร์ที่ควบคุมแล้ว

## การเข้ารหัสและการเก็บไฟล์

- ข้อมูลและไฟล์ถูกเก็บบน **Google Cloud พร้อมการเข้ารหัส**
- ไฟล์ผลลัพธ์ (เช่น MoM PDF) เก็บใน storage แบบ private เข้าถึงผ่านลิงก์ชั่วคราว (signed URL) ที่มีอายุจำกัด ไม่เปิดเป็นสาธารณะ
- ไฟล์เสียงดิบถูกลบทันทีหลังประมวลผลเสร็จ และไฟล์สรุป/transcript ถูกลบภายใน 48 ชั่วโมง

## ร่องรอยการใช้งาน (Audit Log)

PERPOS บันทึก audit log การกระทำสำคัญในระบบ ระบุได้ว่าใครเป็นผู้สั่งงานและเกิดอะไรขึ้น แม้แต่งานที่ประมวลผลโดยระบบเบื้องหลัง (worker) ก็แนบรหัสอ้างอิงผู้สั่งงานจริง (correlation id) เพื่อให้ตรวจสอบย้อนหลังได้

## ความปลอดภัยของบริการเบื้องหลัง (Workers)

งานหนัก (เช่น ถอดเสียง บีบ PDF) ทำงานบนบริการแยก (Google Cloud Run) ที่เรียกใช้ได้เฉพาะผ่านการยืนยันสิทธิ์ (shared secret / Google Cloud IAM) เท่านั้น และบริการเหล่านี้เข้าถึงข้อมูลแบบจำกัดขอบเขต (scoped) ไม่สามารถแตะตารางสิทธิ์หรือข้อมูลผู้ใช้ที่ไม่เกี่ยวข้องได้

## ความเป็นส่วนตัวของข้อมูล

ไฟล์และเสียงของลูกค้าไม่ถูกนำไปฝึกโมเดล AI และ PERPOS ไม่ขายหรือแชร์ข้อมูลส่วนบุคคลกับบุคคลที่สามโดยไม่ได้รับความยินยอม รายละเอียดเพิ่มเติมดูได้ที่หัวข้อ PDPA และความเป็นส่วนตัว หรือสอบถามที่ hello@perpos.ai
